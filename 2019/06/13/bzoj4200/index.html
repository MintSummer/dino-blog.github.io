分析

二合一题。

先考虑老司机的问题，发现不存在向下走，所以必然是一层（$Y$ 相同的称为一层）走完再走下一层。

$f[i][j]$ 表示第 $i$ 层被访问到的第一个点是这一层第 $j$ 个点时的最多点数
$g[i][k]$ 表示第 $i$ 层被访问到的最后一个点是这一层第 $k$ 个点时的最多点数。

对于每一层之间暴力用 $f$ 转移 $g$。讨论 $j$ 和 $k$ 的相对位置：当 $j = k$ 时不能在这一层走，必须往上；否则一定是先从 $j$ 走完 $k$ 的所在方向的相反方向，再走向 $k$。然后从 $g[j][k]$ 向上/左上/右上转移。

状态数是 $O(n)$，时间复杂度 $O(n \times m)$，其中 $m$ 是每一行最大点数。

如果 $(a, b)$ 向左上走能走到 $(c, d)$，则 $b - a = d - c$ 或 $a - b = c - d$；如果 $(a, b)$ 向右上走能走到 $(c, d)$，则 $a + b = c + d$

（本来想要直接开一个 map 用来 dp 的，但是算了算带个 $\log$ 就萎了，然后被离散化搞自闭了）

对于小园丁，因为每条最优策略边要求走的次数下界为 $1$，上界 $\textrm{INF}$，发现就是一个有源汇的上下界最小流（可以戳这里学习）。先建源汇点（称作真实源汇）把所有点连起来，再跑此时的上下界循环流（根据初始流向超级源汇连边，别忘了连真实汇到真实源一条 $\textrm{INF}$ 的边），减去残余网络中真实汇到真实源的最大流即可。

建图只连最优策略边，我们可以先记 $end[i][j]$ 表示第 $i$ 层第 $j$ 个能否作为这一层最后一个走到的点，最终能走到一个最优解。$start[i][j]$ 表示能否使第一个。用 $end$ 得到这一层的 $start$，再用 $start$ 向下更新下层的 $end$ 即可。（就是 dp 的过程倒着退回去，要求每一步转移都只能走最优策略边）
