二次离线莫队 [离线]

发表于 2019-06-12 | 更新于 2019-07-10 | 分类于 算法讲解 | 评论数： | 阅读次数：
本文字数： 11k | 阅读时长 ≈ 18 分钟

分块黑科技啊……真是厉害了，离线分块花样多啊

问题

luogu 4887 第十四分块(前体)

给定序列 $a$，每次查询区间 $[l, r]$ 中满足 $a_i \oplus a_j (l \leqslant i < j \leqslant r)$ 的二进制表示中有 $k$ 个 $1$ 的二元组 $(i, j)$ 的对数。其中 $\oplus$ 表示异或。

（请注意，要求 $i \neq j$）

$n,q \leqslant 10^5$，$a_i \leqslant 2^{14}$

方法

首先可以想到标准莫队，离线排序询问，开个桶记当前区间中这个数出现的次数，每一次移动 $l, r$ 的时候就是 $C_{14}^k$ 的统计答案即可，修改复杂度 $O(n \sqrt n C_{14}^k)$，查询 $O(n)$。

$\max_{k = 0}^{14} C_{14}^k = 3432$，过不去啊嘤嘤嘤。

我们把移动细化，先考虑移动左端点。假定当前的询问区间是 $[ql, r]$，枚举到的区间是 $[l, r]$，其中 $l < ql$，现在要把左端点从 $l$ 移动到 $ql$。

定义点 $i$ 对区间 $[l, r]$ 的贡献为点 $i$ 对于区间 $[l, r]$ 中的数满足题意中条件的对数。

对于 $p \in [l, ql)$，移动 $p$ 后答案要减去 $p$ 对于区间 $[p + 1 , r]$ 的贡献，考虑拆分这个区间，变成 $[1, p]$ 和 $[1, r]$。那么对于每一个 $p$，我们分成两部分讨论

  • $p$ 对于 $[1, p]$ 的贡献都是 $i$ 对于 $[1, i]$ 的贡献的形式，可以预处理并算前缀和

  • $p$ 对于 $[1, r]$ 的贡献是 $O(\sqrt n)$ 个点对于同一个区间的，我们可以把这个点的区间分配给 $r$。在枚举区间 $[1, r]$ 的时候再处理这 $O(\sqrt n)$ 个点的贡献，即将修改操作再次离线。

对于操作一，我们可以直接开个桶，前缀和即可，复杂度 $O(n C_{14}^k)$

对于操作二，发现如果暴力的枚举点统计，还是 $O(n \sqrt n C_{14}^k)$ 的，我们重新定义桶 $B_i$ 表示有 $i$ 这个数对当前这个区间的贡献。因为异或的交换律 $a_i \oplus a_j = t$ 即 $a_j = a_i \oplus t$，那么我们每加入一个数 $a_i$ 到区间，就把 $B_{a_i \oplus t} + 1$。这样我们枚举 $[1, r]$ 时每加入一个数是 $O(C_{14}^k)$ 的，查询 $[1, r]$ 被分配的点区间的贡献是 $O(\sqrt n)$ 的。复杂度 $O(n \sqrt n)$。

先移动 $l$，（此时的区间按照当前的 $r$ 贡献），把 $l$ 变成 $ql$；再移动 $r$（这个区间就对于 $ql$ 贡献了）。

最后统计一下两部分的答案就好了。

如果你也是这样的评测结果：

注意！！！有一个细节，题面中要求 $i \neq j$，那么当 $k = 0$ 时不应当计入 $a_i \oplus a_i = 0$ 的情况。（貌似可以特判掉，但我没有试）

我们发现，右端点移动，是 $r$ 对于区间 $[l, r]$ 的贡献，因为不能相等，所以应当是 $[l, r)$，拆分后就是 $[1, l - 1]$ 和 $[1, r)$。但是左端点的移动是 $l$ 对于 $[l, r]$ 的贡献，不相等后即 $(l, r]$，拆分后是 $[1, l]$ 和 $[1, r]$ 。所以左端点移动的前缀和要算上 $i = j$ 的贡献，右端点不能算。

例题

luogu 5398 [Ynoi2018]GOSICK

给定序列 $a$，询问给定区间 $[l, r]$ 有多少个二元组 $(i, j)$ 满足 $a_i$ 是 $a_j$ 的倍数。

$n, m, a_i \leqslant 10^5$

分析

模仿 luogu 4887 的套路，使用二次离线莫队。

我们现在考虑，区间要从 $[l, r]$ 变成 $[ql, r]$ ，即将左端点从 $l$ 移动到 $ql$，答案变化了多少。

继续拆区间。对于点 $p$ 对于区间 $[p, r]$ 的贡献，拆成 $[1, r]$ 的贡献减去 $[1, p)$ 的贡献。

记 $B_i$ 表示 $i$ 作为当前区间中多少个数的倍数，$Y_i$ 表示因数。每加入一个数，就枚举它的因数（ 约数是 $O(\sqrt n)$ 级别）统计到 $Y$，再枚举它的倍数统计到 $B$。(貌似 $B$ 和 $Y$ 开一个就够了，贡献在同一个上)

但是如果这个数 $x$ 比较小，枚举倍数就太慢了，我们设一个阈值 $S$，令 $S = 32$。如果 $x > S$ 就直接暴力枚举倍数，否则我们令它在 $x$ 的倍数处同统计 $x$ 的贡献。

那么查询 $a_i$ 在当前区间 $[1，r]$ 中有多少个 $a_j(1 \leqslant j \leqslant r)$ 满足 $a_j$ 是 $a_i$ 的因数或者倍数， 即为 $B_{a_i} + Y_{a_i}$加上 $x \leqslant S (x | i)$ 的个数。

但是枚举这样的 $x$ 是 $O(S)$ 的，统计贡献的复杂度就变成 $O(n \sqrt n S)$ 了，过不了，我们试图把他变成 $O(1)$ 的。

设计一个 $01$ 状态 $state$，第 $i$ 位是 $1$ 表示有 $i$ 这个因子，再记 $C_x$ 表示 $x$ 这个数 $\leqslant S$ 的因子的集合。这样如果我们每加入一个数 $x (x \leqslant S)$，就把所有含有 $x$ 的状态（即$x$ 这一位为 $1$）的个数 $Cnt$ 加一，这样查询 $a_i$ 满足小于等于 $S$ 的因子个数就直接是 $Cnt[C_{a_i}]$ 了。

但是总状态是 $2^{32}$ 的，因为答案是累加的，我们考虑把它分成四部分，每 $2^8$ 个状态为一组，这样加入一个数的时候只用枚举 $2^8$ 个状态就可以了，查询查四个部分的和即可。这样操作的次数为加入 $2^8$ 的，查询 $4$。

同样的，模仿 第十四分块 中考虑边界情况。左端点是 $p$ 对 $[p, r]$ 的贡献，拆区间是 $[1, p)$ 和 $[1, r]$；右端点是 $p$ 对 $[l, p]$ 的贡献，拆区间是 $[1, l - 1]$ 和 $[1, p]$。即左端点不算自己，右端点算。

还要注意，直接统计因数和倍数的个数时，二元组 $(i, i)$ 分别作为倍数和因数统计了两边，最后答案减去区间长度即可（每个多算了一个）。
